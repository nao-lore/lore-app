## 手順

### Step 1: 現状調査
1. `ls -la ~/Downloads/lore-project/` で全ファイル確認
2. `ls ~/Downloads/ | grep -v lore-project` でDownloads直下の散らかりを確認
3. 各サブフォルダの中身を確認

### Step 2: 不要ファイルの特定
以下のパターンで不要ファイルを検出:
- 古いバージョン（同じファイルの新版が別の場所にある）
- 中間ファイル（judge-*.json等の一時データ）
- 空フォルダ
- インストーラ（*.dmg, *.pkg）
- 重複ファイル（同じ内容が複数箇所にある）

### Step 3: 画像の分類
- ~/Downloads/ の画像ファイルをReadで中身を確認
- カテゴリ分け: Lore関連 / PHスクショ / note素材 / 不要
- 適切なフォルダに移動

### Step 4: フォルダ構造の整理
推奨構造:
```
~/Downloads/lore-project/
├── SESSION.md
├── 運用マニュアル.md
├── 作業ログ.md
├── 学習ガイド/          ← tech/practical/industry/advanced-guide.html
├── 就活/                ← 応募先リスト、履歴書、応募文、プロフィール等
├── note/                ← 記事原稿
├── Reddit/              ← コメント、投稿文
├── スクレイパー/         ← 求人データ、キャッシュ
├── PH/                  ← PH関連ファイル（手順書、投稿文、スクショ）
└── アセット/            ← 画像、ロゴ、OGP、デモGIF
```

### Step 5: 実行
- 削除対象をリストアップしてユーザーに確認
- 確認後に削除・移動を実行
- SESSION.mdのファイルマップを更新

### Step 6: 整合性チェック
- SESSION.mdのファイルマップと実際のファイルを照合
- CLAUDE.mdの参照パスが有効か確認
- 壊れたリンクがないか確認
